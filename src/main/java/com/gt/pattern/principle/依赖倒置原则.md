~~~~~~~~

    依赖倒置:
        面向接口
        高层模块不依赖低层模块，二者都应该依赖其抽象(不能以来特定的子类)
            这里的高层模块与低层模块: 加入类A中依赖了B 即A类中有调用到B对象的某个方法
        此时高层模块即是A， 低层模块即是B，这个时候最好依赖B的抽象类或者接口
        比如springMvc中controller引用的是接口层而非实际的接口实现类

        细节依赖抽象，而非抽象依赖细节
        
        抽象稳定  细节多变
        抽象设定规范，不涉及具体实现

    依赖传递:
        1.接口传递
            两个接口，其中一个接口的方法参数为另一个接口，并在方法中调用这个接口的方法
        2.构造方法
        3.setter
